# Contribuer à Bugpedia

Merci de votre intérêt ! Ce document résume les attentes pour les issues, les
pull requests et le code.

## Démarrage rapide

1. Forkez le dépôt et clonez-le localement.
2. Suivez la section « Développement local » du `README.md`.
3. Créez une branche : `git checkout -b feat/ma-fonctionnalite`.

## Issues

- Décrivez le **comportement attendu** vs **observé**.
- Pour les bugs API ou UI, joignez si possible : version du navigateur, corps
  de requête, logs.
- Les idées d’évolution sont les bienvenues ; ouvrez une issue « discussion »
  avant un gros refactor.

## Pull requests

- Une PR = une **intention claire** (fix, feature, doc).
- Ajoutez ou mettez à jour les **tests** quand le comportement change.
- Respectez le style existant (formatage, nommage).
- Mettez à jour le `README.md` si le setup ou l’architecture change.

## Code de conduite

Soyez respectueux et constructifs. Le projet vise une communauté ouverte ;
harassment et discrimination ne sont pas tolérés.

## Licence

En contribuant, vous acceptez que vos contributions au **code** soient sous
licence MIT. Les **contenus** (bugs, solutions) publiés sur une instance
publique suivent la politique de données de cette instance (voir `DATA_LICENSE.md`
pour la démo de référence).
